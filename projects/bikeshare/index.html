Capital Bikeshare Dashboard

Interactive ridership analytics across Washington D.C. — weather correlations, seasonal patterns, and station demand.

An interactive Streamlit dashboard analysing the Capital Bikeshare dataset from Washington D.C. The dashboard surfaces ridership patterns across seasons, correlations between weather variables (temperature, humidity, wind speed) and bike demand, and comparative station-level usage trends.

Built with Pandas for data processing and Plotly Express for interactive charts. Streamlit's @st.cache_data decorator ensures fast re-renders when filters change. Key insights include the strong positive correlation between temperature and ridership, summer demand peaks, and a clear weekday-vs-weekend divergence in usage patterns.

Implementation Highlight

Cached data loader + Plotly seasonal ridership chart 
@st.cache_data
def load_data():
    df = pd.read_csv('day.csv')
    df['dteday'] = pd.to_datetime(df['dteday'])
    df['season_label'] = df['season'].map(
        {1: 'Spring', 2: 'Summer', 3: 'Fall', 4: 'Winter'})
    return df

df = load_data()

fig = px.line(
    df, x='dteday', y='cnt',
    color='season_label',
    title='Daily Ridership by Season',
    labels={'cnt': 'Total Rides', 'dteday': 'Date'}
)
st.plotly_chart(fig, use_container_width=True)
