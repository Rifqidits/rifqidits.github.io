Cloud PACS — X-Ray AI

14-class chest pathology detection with Grad-CAM explainability, deployed on GCP at 100% uptime.

A production-deployed AI system for clinical chest radiograph analysis, trained on the NIH ChestX-ray14 dataset (24,999 images). The core model is DenseNet121 fine-tuned for 14-class multi-label pathology detection — Atelectasis, Cardiomegaly, Effusion, Pneumonia, and 10 others — using per-class weighted binary cross-entropy to address severe class imbalance.

A 2-stage preprocessing pipeline combines CLAHE contrast enhancement with featurewise normalization (Keras ImageDataGenerator), improving X-ray feature quality and boosting interpretability metrics by 25% on the held-out test set.

A Grad-CAM explainability module (ResNet50, conv5_block3_out) generates class-activation heatmaps overlaid on X-rays, making model decisions interpretable for clinicians. The full system runs on GCP: Healthcare API (DICOM store), Cloud Functions for DICOM-to-PNG conversion, Cloud Storage for multi-stage pipeline state, and Cloud Run for the OHIF Viewer — enabling cross-hospital access.

Implementation Highlight

Grad-CAM heatmap generation — class-activation maps for clinician-facing explainability 
def make_gradcam_heatmap(img_array, model, last_conv_layer):
    grad_model = tf.keras.Model(
        model.inputs,
        [model.get_layer(last_conv_layer).output, model.output]
    )
    with tf.GradientTape() as tape:
        conv_out, preds = grad_model(img_array)
        loss = preds[:, tf.argmax(preds[0])]

    grads = tape.gradient(loss, conv_out)
    pooled = tf.reduce_mean(grads, axis=(0, 1, 2))
    heatmap = conv_out[0] @ pooled[..., tf.newaxis]
    return tf.nn.relu(tf.squeeze(heatmap)).numpy()
